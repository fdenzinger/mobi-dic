    Results Import¶

    The Results Import tab is the starting point for inspecting DIC outputs inside QGIS. After a processing run has completed, point it to the output directory and scan for results. You can then select individual displacement components, configure their colour scale and range, and load them directly into the QGIS layer tree with one click. From the same tab you can also export layers to Google Earth Pro as georeferenced KMZ overlays, optionally with time slider support for multi-date datasets.

    The workflow is straightforward:

    1. Set the Directory and click Scan for Results to discover all available layers.
    2. Tick the components you need and click ▶ Load Selected into QGIS.
    3. Optionally click 🌍 Export to Google Earth Pro… to create KMZ files from the scanned layers.

    The tab is organised into three collapsible sections covering these steps in order.

    1 · Scan & Select Layers¶

    Discover and select the displacement layers to load. Point the plugin to the output directory, trigger a scan, and choose which components to bring into QGIS.

    Directory & scan¶

    Point Directory to the root folder of a DIC processing run and click Scan for Results. The scan searches recursively for all DIC_*.tif files and populates the layer list. Range computation for the Symbology table is also triggered at this point.

    Component filter¶

    Narrows the layer list to a specific displacement component or result type. After scanning, only components actually found in the directory are offered as options.

    Layer list¶

    Tick the layers you want to load, then click ▶ Load Selected into QGIS. Use the All / None / Invert buttons to manage the selection quickly. Layers are loaded into a named group in the QGIS layer tree.

    Output components¶

    Each DIC run produces one GeoTIFF per displacement component:

    Layer Modes Physical meaning
    DIC_x DEM, Optical East–West displacement. Positive = eastward.
    DIC_y DEM, Optical North–South displacement. Positive = northward.
    DIC_xy DEM, Optical 2-D horizontal magnitude: \(\sqrt{x^2 + y^2}\). Always ≥ 0.
    DIC_dz DEM only Vertical displacement before co-registration offset is restored.
    DIC_z DEM only Vertical displacement after co-registration offset is restored — the physically meaningful value.
    DIC_xyz DEM only 3-D magnitude: \(\sqrt{x^2 + y^2 + z^2}\). Always ≥ 0.
    DIC_az DEM, Optical Displacement azimuth in degrees clockwise from North (0–360°).
    DIC_dip DEM only Displacement dip angle from horizontal (−90° to +90°). Positive = upward.
    DIC_ee DEM, Optical Correlation error — normalised RMS residual from the DFT registration.

    EE (correlation error): What it measures

    EE is the normalised RMS registration error from the DFT cross-correlation algorithm (Guizar-Sicairos et al. 2008):

    \[\mathrm{EE} = \sqrt{\left|1 - \frac{|\mathrm{CC}_\max|^2}{E_\text{ref} \cdot E_\text{sec}}\right|}\]

    where \(\mathrm{CC}_\max\) is the cross-correlation peak value and \(E_\text{ref}\), \(E_\text{sec}\) are the total spectral energies of the reference and secondary image windows. A value of 0 means a perfect correlation peak; 1 means no correlation at all.

    EE always reflects the quality of the 2-D image correlation — for Optical and Webcam mode this is the orthoimage patch; for DEM mode it is the hillshade or orientation image used as the correlation feature. It is therefore an XY-plane quality indicator in all modes and does not directly reflect the vertical displacement quality in DEM results.

    A more representative quality metric is planned for a future release.

    2 · Symbology & Display¶

    Configure the colormap and colour range applied to each component before loading. Each component has sensible defaults but can be overridden per-component or globally with a custom QML file.

    Symbology table¶

    Each component row has four configurable columns:

    Column Description
    Colormap Named colour scale or a custom QML file
    Min / Max Values mapped to the two ends of the colour ramp
    Range Auto — computed from raster data at load time; Fixed — uses the entered Min/Max

    Default colormaps per component:

    Component Default colormap Notes
    X, Y, Z, dZ RdBu (diverging) Range centred at zero
    XY, XYZ Deformation QML Magnitude — always ≥ 0
    AZ AZ bundled QML Fixed 0–360°
    DIP DIP bundled QML Fixed −90° to +90°
    EE YlOrRd Fixed 0–1

    In Auto mode, symmetric components (X, Y, Z, dZ) are automatically centred at zero: bound = max(|p5|, |p95|), range set to [−bound, +bound]. AZ, DIP, and EE always keep their fixed defaults regardless of this setting.

    Use ↺ Reset ranges to Auto to clear any manually entered values and recompute from data.

    Layer opacity¶

    Transparency applied to every raster layer when loaded (default 80 %). Lower values let the basemap show through.

    Custom QML override¶

    Enable Override with custom QML for all layers and browse to a .qml file to apply a single style to every loaded layer, bypassing the per-component table.

    Global settings¶

    The Settings → Results section sets defaults applied on every scan:

    Setting Default Description
    Default range mode Auto Fixed or Auto for all components
    Percentile range 5 % – 95 % Low/high percentile bounds used in Auto mode
    Custom default ranges (table) Per-component Min/Max used in Fixed mode; Reset restores built-in defaults

    Range computation uses GDAL's histogram method — no full raster read is needed; overviews are used where available.

    3 · Google Earth Export¶

    Export selected layers as georeferenced KMZ overlays for Google Earth Pro. Supports time slider animation for multi-date datasets and an embedded colormap legend.

    Select one or more layers in the list and click 🌍 Export to Google Earth Pro… to create a georeferenced KMZ overlay and load it automatically in Google Earth Pro for a quick 3D visualisation with a legend indication.

    Option Description
    TimeSpan tags Adds KML <TimeSpan> tags so Google Earth Pro can animate layers using its time slider. Dates are extracted automatically from output subdirectory names (format YYYYMMDD). Requires at least two dates in the directory name.
    Embed colormap legend Inserts the colour bar as a screen overlay inside each KMZ, positioned in the bottom-left corner with component name, unit, and min/max values.

    Tip

    For a full time series, export all layers at once — each component gets its own KMZ plus a master KML that links them. Use the master KML in Google Earth to keep file sizes small and toggle layers independently.
