    Post-Processing¶

    Re-run vector filtering on existing DIC results without repeating the full correlation. Both filters work independently from the Processing pipeline — you point them at output files from a previous run, tune the thresholds, and produce a new filtered GeoPackage that can be loaded directly into QGIS or exported to Google Earth.

    Two sub-tabs are available, each targeting a different output format:

    Sub-tab Input Algorithm
    Neighbourhood Coherence Filter HDF5 grid Single-pass radius-based coherence filter
    Iterative Neighbour Filter Sparse GeoPackage Multi-pass N-nearest-neighbour similarity filter

    Neighbourhood Coherence Filter (HDF5)¶

    Operates on the interpolated displacement grid stored in HDF5 files. For each vector, all neighbours within a search radius of skip_factor × RadiusMult are found. The vector is rejected if it deviates from the local neighbourhood mean beyond the set direction or amplitude threshold. A global amplitude range and a hard magnitude cap are applied on top. The filter runs in a single pass.

    Relation to the Vector Filter tab

    This is the same algorithm available in the Processing → Vector Filter tab. Post-Processing mode lets you re-run it with different parameters on already-computed HDF5 files without redoing the DIC correlation.

    1 · Input HDF5 Files¶

    Select the HDF5 result files to filter. Files can be added individually or by scanning an entire folder.

    Add one or more HDF5 result files produced by a previous DIC run.

    Button Action
    Add File(s)… Browse and select individual .h5 / .hdf5 files
    Add from Folder… Recursively scan a folder and add all HDF5 files found
    Remove Selected Remove highlighted entries from the list

    Current limitation

    The filter currently processes only the first file in the list. Support for batch processing of multiple files is planned for a future release.

    2 · Filter Parameters¶

    Tune the amplitude, direction, and neighbourhood thresholds that control which vectors are rejected.

    Parameter Default Description
    ThrMin (m) −100 m Lower amplitude bound. Vectors with magnitude below this value are excluded from all outputs.
    ThrMax (m) 100 m Upper amplitude bound. Vectors with magnitude above this value are excluded from all outputs.
    MaxMagnitude (m) 10 m Hard cap applied before neighbourhood comparison. Vectors exceeding this magnitude are removed entirely.
    ColorMax (m) 10 m Upper limit of the plot colourbar. Does not affect which vectors are exported.
    DirThr (°) 180° Direction coherence threshold. Vectors deviating more than this angle from the neighbourhood circular mean direction are rejected. Set to 180° to disable direction filtering.
    AmpThr (m) 5 m Amplitude coherence threshold. Vectors deviating more than this value from the neighbourhood mean amplitude are rejected.
    RadiusMult 5 Controls the search radius: radius = skip_factor × RadiusMult. Larger values include more distant neighbours.
    Downsample 1 Keep every N-th vector in all outputs. Does not affect filtering, only output density.

    3 · Export Options¶

    Choose which output files are written and how displacement arrows are styled when loading into QGIS or exporting to Google Earth.

    Option Description
    Export GeoPackage (.gpkg) Write filtered vectors as a georeferenced vector layer. Required for QGIS loading and Google Earth export. Enabled by default.
    Export text (.txt) Write filtered vectors as a plain-text table.
    Save plot (.png) Render the filtered displacement field overlaid on a hillshade and save as PNG.
    Arrow style Black arrows — uniform black, good for simple overviews. Colored by field — arrows graduated by 2-D or 3-D magnitude.

    Output files are written to the same folder as the input HDF5, named DIC_coherence_filter{run_tag}.gpkg.

    Large vector counts and Google Earth performance

    Google Earth Pro can become slow or unresponsive with more than a few thousand arrows. Use the Downsample parameter to thin the output before exporting — a value of 2–5 typically keeps the KMZ responsive while still conveying the displacement pattern.

    Running and loading results¶

    Click ▶ Run Filter to start. Progress is streamed to the log panel. Once the run completes successfully:

    • ▶ Load in QGIS — loads the output GeoPackage as a vector arrow layer into the QGIS layer tree, coloured by the selected field.
    • 🌍 Export to Google Earth… — saves the output as a KMZ with scaled displacement arrows. You will be prompted for an arrow scale factor (1 = true map-unit length; increase to make short displacements visible).

    Iterative Neighbour Filter (GPKG)¶

    Operates on the sparse point layer exported as a GeoPackage. For each displacement vector, the N nearest neighbours within a maximum search distance are evaluated. Vectors deviating from the neighbourhood mean beyond the set magnitude or direction threshold are rejected. Points with fewer than the minimum number of neighbours within the search distance are always removed. The algorithm processes highest-magnitude vectors first and can run for multiple iterations, each pass refining using only the survivors of the previous one.

    Relation to the Vector Filter tab

    This is the same algorithm available in the Processing → Vector Filter tab (algorithm by Nicolas Oestreicher). Post-Processing mode lets you re-run it on an existing GeoPackage without re-running DIC.

    1 · Input¶

    Browse to the sparse GeoPackage to filter and select which magnitude field to use for neighbourhood comparison.

    Field Description
    GPKG file Browse to the sparse GeoPackage produced by the DIC export step. The file must contain az and xy (and optionally xyz) fields.
    Magnitude field Column used for neighbourhood magnitude comparison. xy = 2-D horizontal magnitude (√(dx²+dy²)); xyz = 3-D magnitude using the restored Z component — DEM mode only.

    2 · Filter Parameters¶

    Set the magnitude and direction thresholds, neighbourhood search distance, and number of filter passes.

    Parameter Default Description
    Magnitude threshold 10 m Maximum allowed magnitude deviation from the neighbourhood mean.
    Direction threshold 15° Maximum allowed azimuth deviation from the neighbourhood circular mean.
    Min neighbours 4 Vectors with fewer neighbours within Max distance are removed regardless of magnitude or direction.
    Max distance 50 m Search radius for finding neighbours. Should match the point spacing of the sparse layer (skip_factor × resolution).
    Iterations 1 Number of filter passes. Each pass refines using only the survivors of the previous one. More passes remove more outliers but can over-thin dense displacement fields.
    Real-time deletion off When enabled, rejected points are removed from the search index immediately during a pass. When disabled, all decisions within a pass use the same snapshot. Real-time mode is order-dependent and slower.

    3 · Export Options¶

    Control how displacement arrows are styled when loading the filtered result into QGIS or exporting to Google Earth.

    Option Description
    Arrow style Black arrows — uniform black. Colored by field — arrows graduated by the selected magnitude field.
    Color by field xy (2-D magnitude) or xyz (3-D magnitude, DEM mode only). Controls arrow colour when loading in QGIS or exporting to Google Earth.

    Output is written to the same folder as the input GeoPackage, named {input_name}_neighbour.gpkg.

    Large vector counts and Google Earth performance

    Google Earth Pro can become slow or unresponsive with dense displacement fields. If the input GeoPackage is large, increase Min neighbours or reduce Max distance to produce a sparser result, or run an additional pass (more iterations) to remove more outliers before exporting.

    Running and loading results¶

    Click ▶ Run Filter to start. Progress is streamed to the log panel. Once complete:

    • ▶ Load in QGIS — loads the output GeoPackage as a vector arrow layer, coloured by the selected field.
    • 🌍 Export to Google Earth… — saves the output as a KMZ. You will be prompted for an arrow scale factor.
