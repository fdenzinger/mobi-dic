    Configuration Reference¶

    All three pipelines share the same YAML structure: io, batch, and dic_params sections.

    io — Input / Output¶

    Key Description
    base_run_tag Prefix for output folder names. Run tags are auto-appended with resolution, window size, and other parameters.
    dem_primary / img_primary Path to the earlier (reference) DEM or image.
    dem_secondary / img_secondary Path to the later (target) DEM or image.
    output_dir Directory where all results are written. Created automatically if it does not exist.

    batch — Resolution & Window Size¶

    Key Description
    base_dem_res / base_img_res Native resolution of the input data in metres per pixel. Use 1.0 for webcam/pixel mode.
    target_resolutions List of resampling factors. [1] = native resolution; [2, 4] runs at half and quarter resolution.
    window_sizes List of DIC correlation window sizes in pixels. Larger windows give smoother but coarser results. E.g. [32, 64, 128].

    dic_params — Algorithm Parameters¶

    Key Type Default Description
    OR 0 or 1 1 DEM only. 1 = orientation image; 0 = hillshade as the correlation feature.
    co_f 0 or 1 1 Enable (1) or disable (0) co-registration (global shift correction before DIC).
    co_os int 2 DFT oversampling factor used during co-registration.
    coreg_method "full" / "bbox" "full" Co-registration window: full = whole image; bbox = rectangular sub-window.
    coreg_bbox [minx, miny, maxx, maxy] — Bounding box for co-registration in the input CRS. Required when coreg_method: "bbox".
    os int 4 DFT oversampling factor for subpixel displacement precision. Higher = more precise but slower.
    skip_factor int 4 Controls the DIC grid step: step = window_size / skip_factor. Higher values produce denser results.
    refcode string "EPSG:2056" EPSG code of the input CRS. Use "pixel" for webcam/non-georeferenced mode.
    auto_reproject_to_utm bool false When true, inputs with a geographic CRS (degrees) are automatically reprojected to UTM before processing.
    export_sparse_gpkg bool false Export the raw (non-interpolated) DIC point results to a GeoPackage file.
    rescale_results bool true When true, output GeoTIFFs are at the resampled resolution; when false, outputs are upscaled back to original resolution.
    align_mode "crop" / "pad" "crop" Webcam only. How to handle inputs of different sizes: crop to the common area, or pad the smaller image with zeros.

    dic_params.VectorFilter — Post-processing¶

    The vector filter runs after DIC to remove outliers and produce final outputs.

    Key Type Description
    Enabled bool Master switch. Set false to skip post-processing entirely.
    OutputTxt bool Write filtered vectors to a CSV text file.
    OutputGpkg bool Write filtered vectors to a GeoPackage.
    OutputPlot bool Save a displacement vector plot as PNG.
    PlotResult bool Show an interactive plot window. Set false for batch/headless runs.
    MaxMagnitude float Exclude vectors with displacement magnitude above this value (map units).
    ThrMin / ThrMax float Direction thresholds: exclude vectors outside this azimuth range (degrees).
    ColorMax float Colorbar upper limit on displacement plots (map units).
    Downsample int Plot every N-th vector arrow (reduces clutter on dense grids).
    BackgroundAlpha float Opacity of the displacement colour overlay (0–1).
    HillshadeTif path Optional background raster for plots (DEM or orthoimage).
    PlotWMS bool Fetch a web map tile background for plots (requires internet).
    WMSLayer string Tile style: "satellite", "terrain", etc.
    BlackArrows bool true = black arrows; false = arrows coloured by magnitude.

    dic_params.AIMask — AI Segmentation Mask (Webcam only)¶

    Removes semantic regions (sky, water, snow, etc.) from results using a pretrained SegFormer model.

    Key Description
    Enabled Master switch.
    ExcludeTypes List of semantic classes to mask out. Options: sky, water, snow, land, vegetation, buildings, mountain.
    Device Inference device: null (auto-detect), "cpu", "cuda", or "mps".
    DilationPixels Expand mask boundaries by N pixels to catch edge fringing.
    DebugSegmentation Save a false-colour segmentation PNG and log detected class IDs.

    Complete example¶

    io:
  base_run_tag: Dischma_Aug22
  dem_primary: data/dem_20220801.tif
  dem_secondary: data/dem_20220901.tif
  output_dir: output/

batch:
  base_dem_res: 1.0
  target_resolutions: [1, 2]
  window_sizes: [64, 128]

dic_params:
  OR: 0
  os: 10
  skip_factor: 4
  co_f: 1
  coreg_method: full
  refcode: EPSG:2056
  export_sparse_gpkg: false
  rescale_results: true
  VectorFilter:
    Enabled: true
    MaxMagnitude: 5.0
    ThrMin: 0.02
    ThrMax: 5.0
    OutputGpkg: true
    OutputPlot: true
    PlotResult: false
    ColorMax: 2.0
    Downsample: 4
    BackgroundAlpha: 0.6
    PlotWMS: false
    BlackArrows: false

    Auto-generated run tags¶

    Output folders are named:

    {base_run_tag}_RES{res}_WIN{wi}_SKF{skip}_OR{or}_COF{co_f}[_COREG{method}]

    Example: Brienz_RES1.0_WIN64_SKF4_OR0_COF1_COREGfull
