    DIC Parameters Tab¶

    Controls the core algorithm settings. These map directly to the batch and dic_params sections of the YAML config.

    Resolution & window size¶

    Parameter Description
    Base resolution (m) Native pixel size; auto-detected from the primary input CRS + GeoTIFF metadata.
    Target resolutions Comma-separated list of resolutions (in metres) to process. The primary input is resampled to each.
    Window sizes Comma-separated list of correlation window sizes (pixels). Larger windows are more robust but coarser.
    Skip factor Grid step between correlation windows (pixels). 1 = dense grid; 2 = every other pixel.

    Correlation feature (DEM mode)¶

    OR flag Feature computed
    0 Hillshade — illumination image from DEM (azimuth 315°, elevation 45° by default)
    1 Orientation — aspect/gradient direction image

    Hillshade works best for smooth topography; orientation is more robust when illumination angles change between acquisitions.

    Co-registration¶

    Parameter Description
    Enable co-registration (co_f) Subtract a bulk shift computed from stable ground before DIC.
    Method full — use the whole image overlap; bbox — restrict to a bounding box drawn on canvas.
    Co-reg bounding box Four spinboxes (Xmin, Ymin, Xmax, Ymax) in the project CRS. Use Draw on Canvas to define interactively.

    Oversampling¶

    os (oversampling factor, default 10) controls sub-pixel precision in the DFT registration step. Higher values give finer sub-pixel resolution at the cost of compute time. Values between 10 and 100 are typical.

    CRS¶

    Detected automatically from the primary input. Shown as an EPSG code (default EPSG:2056 – Swiss LV95 for SLF data). Change it in the Settings dialog if needed.
