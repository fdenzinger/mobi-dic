    CLI & Batch Processing¶

    The Python CLI processes one or more DIC pairs headlessly, driven entirely by a YAML config file. It is the same code the QGIS plugin calls as a subprocess.

    Running from a plugin installation¶

    After installing the plugin via QGIS, the DIC scripts and Python environment are available without any separate setup.

    The easiest way to get the correct invocation is the 📋 Copy Run Command button in the plugin dialog — it resolves the correct Python interpreter and script paths for your system automatically, regardless of QGIS version or installation location, and copies the full command to the clipboard.

    Script location¶

    The plugin is installed inside the QGIS user profile directory. The path contains the QGIS major version number (QGIS3, QGIS4, …) and varies by platform:

    Platform Scripts path
    macOS ~/Library/Application Support/QGIS/QGIS3/profiles/default/python/plugins/slfdic/_dic_scripts/
    Linux ~/.local/share/QGIS/QGIS3/profiles/default/python/plugins/slfdic/_dic_scripts/
    Windows %APPDATA%\QGIS\QGIS3\profiles\default\python\plugins\slfdic\_dic_scripts\

    Tip

    If you are unsure of the exact path, open the QGIS Python console and run QgsApplication.qgisUserDatabaseFilePath() — the plugins folder is a sibling of that file.

    Python interpreter¶

    Platform Path
    macOS / Linux ~/.slf-dic/.pixi/envs/default/bin/python3
    Windows %USERPROFILE%\.slf-dic\.pixi\envs\default\python.exe

    Running from a Pipeline Only installation¶

    With the pipeline-only build, scripts live inside the unzipped distribution folder and the environment is managed by pixi.

    Script location¶

    All processing scripts are in slf-dic-python/ inside the distribution folder.

    Main processing scripts¶

    DEM-based DIC

    cd /path/to/mobi-dic/slf-dic-python
pixi run python run_dic4dem_processing.py --config config_dic4dem.yml

    Optical image DIC

    cd /path/to/mobi-dic/slf-dic-python
pixi run python run_dic4optical_processing.py --config config_dic4optical.yml

    Webcam DIC

    cd /path/to/mobi-dic/slf-dic-python
pixi run python run_dic4webcam_processing.py --config config_dic4webcam.yml

    The --config argument points to a YAML file that controls all processing parameters. See the Configuration Reference for a full description of every option.

    Using conda instead of pixi

    Replace pixi run python with the full path to your conda environment's Python interpreter, e.g. conda run -n mobi-dic python.

    Standalone post-processing scripts¶

    These scripts are called by the plugin's Post-Processing sidebar but can also be run directly.

    Coherence / Vector filter (run_coherence_filter_standalone.py)¶

    Re-runs the VectorFilter on an existing HDF5 result file. Useful for adjusting filter thresholds or re-exporting outputs without rerunning DIC.

    ~/.slf-dic/.pixi/envs/default/bin/python3 \
  ~/Library/Application\ Support/QGIS/QGIS3/profiles/default/python/plugins/slfdic/_dic_scripts/run_coherence_filter_standalone.py \
  --hdf5 /path/to/DIC_result.h5 \
  [--Key value ...]

    ~/.slf-dic/.pixi/envs/default/bin/python3 \
  ~/.local/share/QGIS/QGIS3/profiles/default/python/plugins/slfdic/_dic_scripts/run_coherence_filter_standalone.py \
  --hdf5 /path/to/DIC_result.h5 \
  [--Key value ...]

    %USERPROFILE%\.slf-dic\.pixi\envs\default\python.exe ^
  %APPDATA%\QGIS\QGIS3\profiles\default\python\plugins\slfdic\_dic_scripts\run_coherence_filter_standalone.py ^
  --hdf5 C:\path\to\DIC_result.h5 ^
  [--Key value ...]

    Any additional --Key value arguments are forwarded as VectorFilter parameters (e.g. --MaxMagnitude 3.0 --OutputPlot true).

    Neighbour filter (run_neighbour_filter_standalone.py)¶

    Filters a sparse DIC GeoPackage by comparing each displacement vector to its N nearest neighbours in magnitude and direction.

    ~/.slf-dic/.pixi/envs/default/bin/python3 \
  ~/Library/Application\ Support/QGIS/QGIS3/profiles/default/python/plugins/slfdic/_dic_scripts/run_neighbour_filter_standalone.py \
  --gpkg /path/to/DIC_sparse.gpkg \
  [--output /path/to/filtered.gpkg] \
  [--MagField xy|xyz] \
  [--MagThresh 10.0] \
  [--DirThresh 15.0] \
  [--NNeighbours 4] \
  [--MaxDist 50.0] \
  [--Iterations 1]

    ~/.slf-dic/.pixi/envs/default/bin/python3 \
  ~/.local/share/QGIS/QGIS3/profiles/default/python/plugins/slfdic/_dic_scripts/run_neighbour_filter_standalone.py \
  --gpkg /path/to/DIC_sparse.gpkg \
  [--output /path/to/filtered.gpkg] \
  [--MagField xy|xyz] \
  [--MagThresh 10.0] \
  [--DirThresh 15.0] \
  [--NNeighbours 4] \
  [--MaxDist 50.0] \
  [--Iterations 1]

    %USERPROFILE%\.slf-dic\.pixi\envs\default\python.exe ^
  %APPDATA%\QGIS\QGIS3\profiles\default\python\plugins\slfdic\_dic_scripts\run_neighbour_filter_standalone.py ^
  --gpkg C:\path\to\DIC_sparse.gpkg ^
  [--output C:\path\to\filtered.gpkg] ^
  [--MagField xy] ^
  [--MagThresh 10.0] ^
  [--DirThresh 15.0] ^
  [--NNeighbours 4] ^
  [--MaxDist 50.0] ^
  [--Iterations 1]
    Argument Default Description
    --gpkg (required) Input sparse GeoPackage from the DIC pipeline
    --output auto Output path; defaults to <input>_filtered.gpkg
    --MagField xy Magnitude field to compare: xy (2-D) or xyz (3-D)
    --MagThresh 10.0 Maximum allowed magnitude difference between neighbours (map units / pixels)
    --DirThresh 15.0 Maximum allowed azimuth difference between neighbours (degrees)
    --NNeighbours 4 Number of nearest neighbours to compare
    --MaxDist 50.0 Maximum search radius for neighbours (map units / pixels)
    --Iterations 1 Number of filter passes

    Logging¶

    Each run writes a log file run_log_<timestamp>.txt in the output directory. The log includes all parameter values, per-window timing, and any warnings.
